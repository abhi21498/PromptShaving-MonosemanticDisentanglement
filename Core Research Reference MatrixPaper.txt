Algorithmic Formulations (Mathematical Baselines)A. Tri-Axis Memory Retrieval ScoringPopularized by Park et al., this formula calculates the final viability score of a memory slot by blending modern vector space measurements with traditional time-decay functions and static importance scores.$$S_{\text{retrieval}} = \alpha \cdot S_{\text{recency}} + \beta \cdot S_{\text{importance}} + \gamma \cdot S_{\text{relevance}}$$Where:$S_{\text{recency}} = \lambda^{(t - t_{\text{last}})}$: Exponential time-decay function where $\lambda \in [0, 1]$ represents the decay constant, $t$ is current simulation time, and $t_{\text{last}}$ is the timestamp of the last access/update.$S_{\text{importance}}$: Static or LLM-predicted utility score ($1$ to $10$) captured at the extraction boundary.$S_{\text{relevance}}$: Vector cosine similarity between the embedded query vector $\vec{q}$ and the memory material vector $\vec{m}$:$$S_{\text{relevance}} = \frac{\vec{q} \cdot \vec{m}}{\|\vec{q}\| \|\vec{m}\|}$$B. Hybrid Retrieval Score Fusion (RRF Baseline)To blend vector-space (pgvector) candidates with strict keyword overlap (BM25), Reciprocal Rank Fusion (RRF) ensures stable sorting:$$RRF\_Score(d \in D) = \sum_{m \in M} \frac{1}{k + r_m(d)}$$Where $M$ is the set of retrieval strategies (keyword, vector), $r_m(d)$ is the rank assigned to document $d$ by strategy $m$, and $k$ is a smoothing constant (typically $k \approx 60$).4.
