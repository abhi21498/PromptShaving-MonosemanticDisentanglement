Sequential governance checks per atom (tenant → status → consent → retention → sensitivity → score)

Sequential Checks (1→6)

  • 1. Tenant/User Scope (defense-in-depth)
  • 2. Status = Active (deleted/archived blocked)
  • 3. Consent = Granted (withdrawn/expired → BLOCK)
  • 4. Retention Window (exempt if legal_hold/pin/protect)
  • 5. Sensitivity Gate (opt-in: high + enabled → BLOCK)
  • 6. Score Threshold (opt-in: score < threshold → BLOCK)

Block Reasons (per-atom)

  • • BLOCK_WRONG_TENANT
  • • BLOCK_DELETED / BLOCK_ARCHIVED / BLOCK_INACTIVE
  • • BLOCK_CONSENT_WITHDRAWN
  • • BLOCK_EXPIRED (retention)
  • • BLOCK_SENSITIVE (opt-in)
  • • BLOCK_LOW_CONFIDENCE (opt-in)
  • • BLOCK_LEGAL_HOLD (governance override)

ALLOW Criteria

  • • All 6 checks PASS
  • • admission_decision = "ALLOW"
  • • admission_reason = "relevant, active, consent-granted, tenant-scoped"
  • • retrieval_score preserved
  • • score_breakdown in trace
  • → Context Composer → LLM

Governance State (per atom)

  • • consent: granted/withdrawn/expired/not_required
  • • legal_hold: true/false + reason
  • • pinned: true → exempt decay/archive
  • • protected: true → exempt retention
  • • retention: policy + expires_at + evaluated_at
  • • sensitivity: low/medium/high

Audit Trail

  • • context_admission_blocked (per atom)
  • • metadata: {blocked_count, decisions, blocked_ids}
  • • memory_retrieved (for ALLOW set)
  • • Per-atom trace in MemoryUsageTrace
  • • admission_counts: {ALLOW, BLOCK_*}

Config Flags (opt-in)
