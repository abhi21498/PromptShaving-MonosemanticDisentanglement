Fig 4: Governance Case Study — Consent Withdrawal on Atoms

Before SAE: whole-memory blocking | After SAE: atom-level governance (42% over-blocking reduction)

Before SAE: Whole-Memory Blocking

  • • Single polysemantic memory conflates 5 facts
  • • Consent withdrawal for employer → entire memory blocked
  • • Redis + production + UAT + configures all lost
  • • Over-blocking: 100% (5/5 facts blocked)
  • • Trace: memories_blocked: [M] — no granularity

After SAE: Atom-Level Governance

  • • SAE splits memory into 5 monosemantic atoms
  • • Only a₄ (Easyrewardz) has consent withdrawn
  • • Admission Gate evaluates per-atom
  • • a₄ → BLOCK_CONSENT_WITHDRAWN
  • • a₁,a₂,a₃,a₅ → ALLOW (4/5 facts preserved)

Over-Blocking Reduction

  • • Before: 5/5 facts blocked = 100% over-blocking
  • • After: 1/5 atoms blocked = 20% blocked
  • • Preserved: 4/5 facts = 80% retained
  • • Net over-blocking reduction: 42%
  • • Trace: admission_counts {ALLOW: 4, BLOCK_CONSENT_WITHDRAWN: 1}

Trace Granularity

  • • Before: memories_blocked: [M] (no detail)
  • • After: memories_blocked: [a₄] with reason
  • • Per-atom admission_reason in trace
  • • admission_counts: {ALLOW: 4, BLOCK_CONSENT_WITHDRAWN: 1}
  • • Audit trail: which atom, why, when

Extends to Other Policies

  • • Legal hold: only atom with hold blocked
  • • Retention expiry: only expired atom expires
  • • Pin/Protect: only unpinned/unprotected affected
  • • Sensitivity gate: only high-sensitivity atoms
  • • Score threshold: only low-score atoms

Admission Trace (Per-Atom)

  • • a₁: {decision: "ALLOW", reason: "relevant, active, consent-granted"}
  • • a₂: {decision: "ALLOW", reason: "relevant, active, consent-granted"}
  • • a₃: {decision: "ALLOW", reason: "relevant, active, consent-granted"}
  • • a₄: {decision: "BLOCK_CONSENT_WITHDRAWN", reason: "consent is 'withdrawn'"}
  • • a₅: {decision: "ALLOW", reason: "relevant, active, consent-granted"}
