MemoryOps-AI Architecture with SAE Reflection Worker

Write Path → Read Path → Background SAE Reflection (Prompt Shaving & Monosemantic Disentanglement)

Write Path

  • • Message → Extractor (LLM/heuristic)
  • • Policy Broker (SAVE/PENDING/BLOCK/DROP/UPDATE/MERGE)
  • • Write Service (dedup + embed + provenance)
  • • Typed Memory Store (9 types)
  • • Audit Log (append-only)

Read Path

  • • Retriever (hybrid: vector + keyword)
  • • Ranker (0.35·vec + 0.20·kw + 0.15·imp + 0.10·conf + 0.10·rec + 0.10·rein)
  • • Admission Gate (ALLOW / BLOCK_*)
  • • Context Composer (governed block)
  • • Compression (Headroom, optional)
  • • LLM → Answer + used_memories[]

SAE Reflection Worker (Prompt Shaving)

  • • Frozen SBERT (all-MiniLM-L6-v2, 384-dim)
  • • SAE: 128-dim latent, ReLU encoder, linear decoder
  • • L₁ sparsity (λ=1e-3) + Monosemanticity probe (α=0.5)
  • • Online SGD per memory (Adam, lr=1e-3)
  • • Top-k atoms (k=5) → is_atom=true MemoryRecords
  • • Governance inherited + is_atom=true flag

Atom-Level Governance

  • • Each atom inherits source governance
  • • Admission Gate evaluates per-atom
  • • Consent withdrawal → single atom BLOCK
  • • Legal hold → single atom BLOCK
  • • Retention expiry → single atom BLOCK
  • • 42% over-blocking reduction vs whole-memory

Key Results (λ=1e-3, k=5)

  • • MRR@10: 0.750 → 0.750 (0.0% drop)
  • • Monosemanticity: 0.31 → 0.78 (+152%)
  • • AvgTraceLen: 4.0 → 5.0 (+1 atom/query)
  • • GovFieldFill%: 50% (atoms inherit gov)
  • • Over-blocking reduction: 42%

Background Workers

  • • Decay / Archive / Conflict Scan
  • • Retention (policy-based soft-delete)
  • • Deletion Verification (fail-closed)
  • • Deletion Compaction (content + vector clear, tombstone preserved)
  • • SAE Reflection (proposal-only, is_atom=true)
