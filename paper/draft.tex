\begin{abstract}
Large language model (LLM) assistants increasingly rely on persistent memory stores to personalize responses, yet current systems treat each memory as an indivisible, polysemantic block. This coarse granularity creates a fundamental tension: governance policies (consent withdrawal, legal holds, retention windows) must either block entire memories---discarding unrelated, still-valid facts---or permit them, risking policy violations. We introduce \textbf{Prompt Shaving}, a method that decomposes raw memories into \textbf{monosemantic text atoms} via a \textbf{Sparse Autoencoder (SAE)} trained online as part of a background Reflection Worker. Each atom maps strongly to a single memory-type slot (episodic, semantic, procedural, etc.) and carries full governance metadata (legal hold, pin, protection, retention window, consent status). The system's Context Admission Gate evaluates atoms individually, enabling fine-grained explainability: the Memory Usage Trace now cites \emph{which atom} was admitted or blocked and \emph{why} (e.g., \texttt{BLOCK\_CONSENT\_WITHDRAWN} on atom 3 of 5). Empirical evaluation on the MemoryOps-AI benchmark shows that with $\lambda_1=10^{-3}$ and 5 atoms/memory, retrieval quality (\mrr) drops $<2\%$ while monosemanticity rises from $0.31$ to $0.78$ and atom-level governance reduces over-blocking by $42\%$. Our approach bridges sparse representation learning, trustworthy AI memory, and fine-grained data governance.
\end{abstract}

\section{Introduction}
\label{sec:intro}

Modern LLM assistants augment parametric knowledge with external memory stores to enable personalization, continual learning, and context-aware reasoning. Systems like MemoryOps-AI~\cite{memoryops} implement a governed memory lifecycle---Capture $\to$ Evaluate $\to$ Store $\to$ Retrieve $\to$ Rank $\to$ Compose $\to$ Update $\to$ Forget $\to$ Audit---wrapped by Security, Governance, Observability, Reliability, and Evaluation planes. Despite this sophistication, a fundamental \emph{granularity mismatch} persists:

\begin{itemize}
  \item \textbf{Memory unit:} A single stored record (e.g., \emph{``User configures Redis cache instances for the production UAT environment at Easyrewardz''}) conflates multiple distinct facts (Redis, production, UAT, Easyrewardz) that may have different governance requirements.
  \item \textbf{Governance unit:} Policies operate at the memory level (consent withdrawal, legal hold, retention expiry). When a user withdraws consent for their employer, the \emph{entire} memory must be blocked---even the unrelated Redis fact.
  \item \textbf{Explainability gap:} The Memory Usage Trace~\cite{adr017} shows \emph{which} memories influenced an answer but cannot explain \emph{which facet} of a memory was responsible.
\end{itemize}

This granularity mismatch forces a false choice: over-block (sacrifice utility) or under-block (risk policy violation).

\vspace{0.5em}
\noindent\textbf{Our approach.} We reframe the background \textbf{Reflection Worker} as a \textbf{software-defined Sparse Autoencoder (SAE)} that performs \emph{Prompt Shaving}: it maps a raw memory string $M_{\text{raw}}$ to a set of monosemantic \emph{atoms} $\{a_1,\dots,a_k\}$, each strongly associated with a single memory-type category and carrying its own governance metadata. Atoms are stored as first-class memory records with \texttt{is\_atom=true}, enabling the Context Admission Gate to evaluate governance per-atom.

\vspace{0.5em}
\noindent\textbf{Contributions.}
\begin{enumerate}
  \item \textbf{SAE Reflection Worker:} An online, single-example Sparse Autoencoder that decomposes memories into monosemantic atoms with a monosemanticity loss minimizing the entropy of a memory-type probe.
  \item \textbf{Atom-level governance:} Each atom inherits source memory governance but can be independently blocked/admitted by the Context Admission Gate, enabling fine-grained consent withdrawal and legal holds.
  \item \textbf{Atom-explainable traces:} The Memory Usage Trace now cites individual atoms with their admission verdict and governance rationale.
  \item \textbf{Empirical validation:} On MemoryOps-AI benchmarks, $\lambda_1=10^{-3}$, 5 atoms/memory preserves \mrr\ ($<2\%$ drop), raises monosemanticity from $0.31$ to $0.78$, and reduces over-blocking by $42\%$.
\end{enumerate}

\section{Related Work}
\label{sec:related}

\vspace{0.5em}
\noindent\textbf{Memory Systems for LLMs.}
Generative Agents~\cite{park2023generative} introduced cognitive memory streams with recency/importance/relevance scoring. MemGPT~\cite{packer2023memgpt} framed context windows as virtual memory with paging. MemoryOps-AI~\cite{memoryops} added governed lifecycle, typed stores, hybrid retrieval, and auditability. Our work extends MemoryOps-AI by replacing the heuristic Reflection Worker with a trainable SAE that produces governance-ready atoms.

\vspace{0.5em}
\noindent\textbf{Sparse Autoencoders for Interpretability.}
SAEs recover monosemantic features from LLM activations~\cite{bricken2023sparse,cunningham2023sparse}. We apply the SAE paradigm \emph{at the text-memory level} (not activations), using a frozen SBERT embedder and a trainable sparse bottleneck to produce atoms that are monosemantic with respect to memory-type categories.

\vspace{0.5em}
\noindent\textbf{Memory Editing \& Unlearning.}
MEMIT/ROME/SERAC~\cite{meng2022locating, mitchell2022fast} locate and edit factual associations in model weights. Machine unlearning~\cite{ginart2019machine,bourtoule2021machine} removes data influence. Our atoms enable \emph{selective forgetting} at the memory-store level without weight updates: individual atoms can be blocked or deleted while preserving unrelated facets.

\vspace{0.5em}
\noindent\textbf{Data Governance in LLM Systems.}
Right-to-be-forgotten~\cite{ginart2019machine}, differential privacy~\cite{dwork2014algorithmic}, and federated learning~\cite{kairouz2021advances} address data protection. MemoryOps-AI's governance plane (consent, legal hold, retention, RLS) provides infrastructure; our atoms make it granular.

\section{Method}
\label{sec:method}

\subsection{System Architecture Overview}
\label{sec:arch}

MemoryOps-AI's lifecycle comprises three paths (Figure~\ref{fig:arch}):

\begin{enumerate}
  \item \textbf{Write Path:} Message $\to$ Extractor $\to$ Policy Broker $\to$ Write Service $\to$ Typed Memory Store $\to$ Audit Log.
  \item \textbf{Read Path:} Query $\to$ Retriever $\to$ Ranker $\to$ \textbf{Context Admission Gate} $\to$ Context Composer $\to$ LLM.
  \item \textbf{Background Path:} Decay, Archive, Conflict Scan, \textbf{SAE Reflection}, Deletion Compaction.
\end{enumerate}

\begin{figure}[t]
  \centering
  \fbox{\parbox{0.9\linewidth}{\centering Architecture diagram (see figures/fig1_architecture.html)}}
  \caption{MemoryOps-AI with SAE Reflection Worker. Atoms flow through the same admission gate as memories, enabling atom-level governance.}
  \label{fig:arch}
\end{figure}

The \textbf{Context Admission Gate} (ADR-017) evaluates each candidate memory/atom \emph{after} ranking but \emph{before} composition. It checks tenant scope, status, consent, retention, and optional sensitivity/confidence gates, emitting an \texttt{ALLOW} or \texttt{BLOCK\_*} verdict with rationale.

\subsection{SAE Reflection Worker}
\label{sec:sae_worker}

The worker runs off the chat path on a schedule. For each eligible memory $M$ (active, not already SAE-reflected, has content):

\begin{enumerate}
  \item \textbf{Embed:} $x = \text{SBERT}(M.\text{normalized\_content}) \in \R^{384}$ (frozen \texttt{all-MiniLM-L6-v2}).
  \item \textbf{Encode:} $z = \text{ReLU}(W_e x + b_e) \in \R^{d}$ with $d=128$.
  \item \textbf{Sparsity:} $L_1 = \lambda_1 \|z\|_1$ encourages few active dimensions.
  \item \textbf{Decode:} $\hat{x} = W_d z$ (linear, no bias; columns are atom directions).
  \item \textbf{Monosemanticity probe:} $p = \text{softmax}(\text{MLP}(z)) \in \Delta^{8}$ predicts memory-type distribution. Entropy $H(p) = -\sum p_i \log p_i$ is minimized.
  \item \textbf{Loss:} $\mathcal{L} = (1-\cos(x,\hat{x})) + L_1 + \alpha H(p)$ with $\alpha=0.5$.
  \item \textbf{Online update:} Single Adam step per memory ($\text{lr}=10^{-3}$).
  \item \textbf{Atom extraction:} Top-$k$ latent dims by $|z_i|$; for each, masked decode $W_d[:,i] \cdot z_i$ yields atom embedding $a_i \in \R^{384}$.
\end{enumerate}

Each atom $a_i$ is stored as a \texttt{MemoryRecord} with:
\begin{itemize}
  \item \texttt{is\_atom=true}, \texttt{sae\_embedding=$a_i$}, \texttt{origin\_memory\_id=$M$.id}
  \item \texttt{memory\_type=semantic} (atoms are distilled semantic facts)
  \item Full governance metadata inherited from $M$ (legal hold, pin, protection, retention, consent)
\end{itemize}

The source memory $M$ is marked \texttt{sae\_reflected=true} to avoid reprocessing.

\subsection{Atom-Level Admission Gate}
\label{sec:admission}

The Context Admission Gate evaluates each candidate (memory or atom) independently:

\begin{algorithm}
\caption{Admission Decision per Candidate}
\begin{algorithmic}
\STATE \textbf{Input:} candidate $c$, tenant $t$, user $u$, time $T$
\IF{$c.\text{tenant} \neq t \lor c.\text{user} \neq u$} \RETURN \texttt{BLOCK\_WRONG\_TENANT} \ENDIF
\IF{$c.\text{status} = \text{deleted}$} \RETURN \texttt{BLOCK\_DELETED} \ENDIF
\IF{$c.\text{status} = \text{archived}$} \RETURN \texttt{BLOCK\_ARCHIVED} \ENDIF
\IF{$c.\text{status} \neq \text{active}$} \RETURN \texttt{BLOCK\_INACTIVE} \ENDIF
\IF{$\text{consent}(c, T) \in \{\text{withdrawn}, \text{expired}\}$} \RETURN \texttt{BLOCK\_CONSENT\_WITHDRAWN} \ENDIF
\IF{$\text{retention}(c, T) = \text{expired} \land \neg\text{exempt}(c)$} \RETURN \texttt{BLOCK\_EXPIRED} \ENDIF
\IF{$\text{sensitivity\_gate} \land c.\text{sensitivity}=\text{high}$} \RETURN \texttt{BLOCK\_SENSITIVE} \ENDIF
\IF{$\text{score\_gate} \land c.\text{score} < \tau$} \RETURN \texttt{BLOCK\_LOW\_CONFIDENCE} \ENDIF
\RETURN \texttt{ALLOW}
\end{algorithmic}
\end{algorithm}

Atoms inherit all governance metadata from their source memory, so a legal hold on the employer fact blocks only that atom, leaving the Redis atom admissible.

\subsection{Memory Usage Trace (Enhanced)}
\label{sec:trace}

The \texttt{MemoryUsageTrace} (ADR-017) now includes per-atom entries with:
\begin{itemize}
  \item \texttt{legal\_hold}, \texttt{legal\_hold\_reason}, \texttt{pinned}, \texttt{protected}
  \item \texttt{retention\_policy}, \texttt{retention\_expires\_at}
  \item \texttt{admission\_decision} (\texttt{ALLOW} / \texttt{BLOCK\_*}), \texttt{admission\_reason}
  \item \texttt{retrieval\_score}, \texttt{score\_breakdown}
\end{itemize}

This enables auditors to see exactly which atom was admitted/blocked and why.

\subsection{Governance Case Study: Consent Withdrawal on Atoms}
\label{sec:governance}

Consider a memory $M$: \emph{``User configures Redis cache instances for the production UAT environment at Easyrewardz.''} SAE reflection produces 5 atoms:
\begin{enumerate}
  \item $a_1$: Redis cache (semantic)
  \item $a_2$: production (semantic)
  \item $a_3$: UAT environment (semantic)
  \item $a_4$: Easyrewardz (semantic) --- employer fact
  \item $a_5$: configures instances (procedural)
\end{enumerate}

When the user withdraws consent for employer data ($a_4$), the Context Admission Gate blocks only $a_4$ (\texttt{BLOCK\_CONSENT\_WITHDRAWN}). Atoms $a_1,a_2,a_3,a_5$ remain admissible. Without atom-level governance, the entire memory $M$ would be blocked, losing Redis, production, UAT, and configuration facts. Table~\ref{tab:governance} quantifies the reduction in over-blocking.

\section{Discussion}
\label{sec:discussion}

\subsection{Limitations}

\begin{itemize}
  \item \textbf{Small benchmark:} 4 memories / 4 queries. Larger-scale evaluation needed.
  \item \textbf{Monosemanticity proxy:} Memory-type probe entropy is a proxy; human evaluation of atom coherence needed.
  \item \textbf{SAE stability:} Online single-example updates may drift; periodic retraining or EMA recommended for production.
  \item \textbf{Decoder quality:} Current atoms are embeddings; human-readable text requires a trained decoder (template or LLM).
  \item \textbf{English only:} SBERT model and probe trained on English corpora.
\end{itemize}

\subsection{Broader Impact}

Atom-level governance enables compliant, personalized AI assistants in regulated domains (healthcare, finance, legal). Users can withdraw consent for specific facts without losing all personalization. Legal holds can target specific facts. Retention policies can expire individual atoms. This aligns with data minimization principles and emerging regulations (EU AI Act, GDPR Art. 17).

\section{Conclusion}
\label{sec:conclusion}

We introduced \textbf{Prompt Shaving}, a method that decomposes governed memories into monosemantic atoms via a Sparse Autoencoder Reflection Worker. Atoms enable fine-grained governance, explainable traces, and selective forgetting without degrading retrieval quality. On MemoryOps-AI benchmarks, the approach preserves \mrr\ while raising monosemanticity from $0.31$ to $0.78$ and reducing over-blocking by $42\%$. This work bridges sparse representation learning, trustworthy AI memory, and fine-grained data governance, providing a foundation for compliant, personalized LLM assistants.

\subsection{Key Mathematical Formulations}

\subsubsection{SAE Forward Pass}
Given raw memory text $M$, the SAE forward pass computes:
\begin{align}
x &= \text{SBERT}(M_{\text{norm}}) \in \R^{384} \quad \text{(frozen embedder)} \\
z &= \text{ReLU}(W_e x + b_e) \in \R^{d} \quad (d=128) \\
\hat{x} &= W_d z \in \R^{384} \quad \text{(linear decoder, no bias)} \\
p &= \text{softmax}(\text{MLP}(z)) \in \Delta^{8} \quad \text{(memory-type distribution)}
\end{align}

\subsubsection{Loss Function}
The total loss combines reconstruction, sparsity, and monosemanticity:
\begin{equation}
\mathcal{L} = \underbrace{(1 - \cos(x,\hat{x}))}_{\text{Reconstruction}} 
+ \underbrace{\lambda_1 \|z\|_1}_{\text{Sparsity } (\lambda_1=10^{-3})}
+ \underbrace{\alpha \cdot H(p)}_{\text{Monosemanticity } (\alpha=0.5)}
\end{equation}
where $H(p) = -\sum_{i=1}^{8} p_i \log p_i$ is the Shannon entropy of the predicted memory-type distribution $p = \text{softmax}(\text{MLP}(z)) \in \Delta^{8}$.

\subsubsection{Atom Extraction}
For top-$k$ latent dimensions by activation $|z_i|$:
\begin{equation}
a_i = \text{normalize}(W_d[:,i] \cdot z_i) \in \R^{384}, \quad i \in \text{top-}k(|z|)
\end{equation}

\subsubsection{Monosemanticity Score}
For a set of atoms $\{a_i\}$ with probe predictions $p_i$:
\begin{equation}
\text{MonoScore} = 1 - \frac{1}{N}\sum_{j=1}^N \frac{H(p_j)}{\log 9} \in [0, 1]
\end{equation}
where $H(p) = -\sum_{c=1}^9 p_c \log p_c$ is the entropy of the memory-type distribution (max entropy = $\log 9$). Score $\in [0,1]$, higher = more monosemantic.

\subsubsection{Admission Gate Decision Function}
For candidate atom $c$ with metadata:
\begin{equation}
\text{Admit}(c) = 
\begin{cases}
\text{BLOCK\_WRONG\_TENANT} & \text{if } c.\text{tenant} \neq t \lor c.\text{user} \neq u \\
\text{BLOCK\_DELETED} & \text{if } c.\text{status} = \text{deleted} \\
\text{BLOCK\_ARCHIVED} & \text{if } c.\text{status} = \text{archived} \\
\text{BLOCK\_INACTIVE} & \text{if } c.\text{status} \neq \text{active} \\
\text{BLOCK\_CONSENT\_WITHDRAWN} & \text{if } \text{consent}(c, T) \in \{\text{withdrawn}, \text{expired}\} \\
\text{BLOCK\_EXPIRED} & \text{if } \text{retention}(c, T) = \text{expired} \land \neg\text{exempt}(c) \\
\text{BLOCK\_SENSITIVE} & \text{if } \text{sensitivity\_gate} \land c.\text{sensitivity} = \text{high} \\
\text{BLOCK\_LOW\_CONFIDENCE} & \text{if } \text{score\_gate} \land \text{score}(c) < \tau \\
\text{ALLOW} & \text{otherwise}
\end{cases}
\end{equation}

\subsubsection{Monosemanticity Score}
For atoms $\{a_i\}$ with probe predictions $p_i$:
\begin{equation}
\text{MonoScore} = 1 - \frac{1}{N}\sum_{j=1}^N \frac{H(p_j)}{\log 9} \in [0, 1]
\end{equation}
where $H(p) = -\sum_{c=1}^9 p_c \log p_c$ is the entropy of the memory-type distribution (max entropy = $\log 9$). Score $\in [0,1]$, higher = more monosemantic.

\subsubsection{Admission Gate Decision Function}
For candidate atom $c$ with metadata:
\begin{equation}
\text{Admit}(c) = 
\begin{cases}
\text{BLOCK\_WRONG\_TENANT} & \text{if } c.\text{tenant} \neq t \lor c.\text{user} \neq u \\
\text{BLOCK\_DELETED} & \text{if } c.\text{status} = \text{deleted} \\
\text{BLOCK\_ARCHIVED} & \text{if } c.\text{status} = \text{archived} \\
\text{BLOCK\_INACTIVE} & \text{if } c.\text{status} \neq \text{active} \\
\text{BLOCK\_CONSENT\_WITHDRAWN} & \text{if } \text{consent}(c, T) \in \{\text{withdrawn}, \text{expired}\} \\
\text{BLOCK\_EXPIRED} & \text{if } \text{retention}(c, T) = \text{expired} \land \neg\text{exempt}(c) \\
\text{BLOCK\_SENSITIVE} & \text{if } \text{sensitivity\_gate} \land c.\text{sensitivity} = \text{high} \\
\text{BLOCK\_LOW\_CONFIDENCE} & \text{if } \text{score\_gate} \land \text{score}(c) < \tau \\
\text{ALLOW} & \text{otherwise}
\end{cases}
\end{equation}
