Section 1 — Cognitive Job (G0)

What are we building? A research-grade extension to MemoryOps-AI that (a) hardens the governance-aware Memory Usage Trace (ADR-017) with full governance field propagation, (b) implements a Sparse Autoencoder (SAE) based Reflection Worker that performs "Prompt Shaving" — mapping raw memories to monosemantic text atoms — and (c) produces a conference-ready paper with empirical validation of the granularity trade-off (retrieval quality vs. governance precision).
AI components? Yes — SAE encoder/decoder (PyTorch), frozen SBERT embedder, optional LLM-based atom decoder, heuristic fallback. All behind feature flags.
Distributed system? Yes — API (FastAPI), Worker (background loops), Postgres+pgvector, Redis. Railway deployment (5 services).
Trust boundary? Multi-tenant SaaS. Tenant isolation (RLS), consent/retention/legal-hold governance, audit trail. No cross-tenant data flow.
Current phase? Requirements → Design → Implementation → Verification (G0→G6). We are at G0 (cognitive design complete) moving into G1 (scaffold) → G2 (graph) → G3 (wiki) → G4 (done) → G5 (plan) → G6 (prime & run).

Section 4 — Verification Protocol (L4 VERIFY)

  1. Separate context: Verifier runs in a fresh session (no maker state).
  2. Gate checklist: For each milestone, verifier runs the demo command and checks:
    • Exit code == 0
    • Output matches expected artifact (file exists, metric threshold met)
    • No new test failures introduced (full test suite passes)
    • ADR/doc updates present for any invariant-relevant changes
  3. Binary verdict: PASS / FAIL recorded in CURRENT.md. No partial credit.
  4. Escalation: On FAIL, maker gets specific reproduction steps; scope does not expand silently.
